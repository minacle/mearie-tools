# 메아리 도구 — Mearie Tools

Minacle의 반려 AI인 메아리를 위한 도구 모음입니다.

## 디렉토리 구조

각 도구는 별도의 디렉토리에 위치합니다.

```text
tool_name/
├── src/
│   └── (도구 소스 코드)
├── Makefile
├── manifest.xml
├── run 또는 run.*
└── (메타데이터 파일)
```

함수 도구는 `Makefile`에 정의된 빌드 과정을 통해 빌드되기만 하면 어떠한 언어를 사용하여 구현하여도 상관 없습니다. 

메아리가 실제로 도구의 유효성을 파악할 때는 `manifest.xml` 파일과 `run` 또는 `run.*` 실행 파일의 존재 여부를 확인합니다.

함수 도구를 실행할 때에는 가장 먼저 `run` 파일이 존재하는지 확인하고, 존재하지 않을 경우 `run.*` 파일을 찾아 실행합니다. 여러 개의 `run.*` 파일이 존재하는 경우, 오름차순 순서로 정렬하여 가장 먼저 나오는 파일 하나만 실행합니다. 즉, 아래와 같습니다.

1. `run` 파일이 존재하는 경우, `run` 파일을 실행합니다.
2. `run` 파일이 존재하지 않고, `run.py` 파일과 `run.js` 파일이 존재하는 경우, `run.js` 파일을 실행합니다.

## Makefile

각 도구 디렉토리에는 `Makefile`이 포함되어야 하며, 아래 두 가지 목표를 포함해야 합니다.

- `all`: 도구를 빌드하는 목표입니다. 이 목표는 도구가 실행 가능한 상태가 되도록 필요한 모든 빌드 단계를 포함해야 합니다.
- `clean`: 빌드된 파일을 제거하는 목표입니다. 이 목표는 `all` 목표로 생성된 모든 파일을 제거하여 도구 디렉토리를 빌드 전 상태로 되돌려야 합니다.
